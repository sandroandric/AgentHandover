## **AgentHandover**

**Goal:** Build a local, always‑on “apprentice” subsystem that **silently observes** your normal day‑to‑day laptop work (without taking actions yet), **learns your workflows**, and produces robust, maintainable **semantic SOPs** that agents like OpenClaw can later execute safely.


## **1\) Mandatory logging (traceability)**

* **Log every completed action** in `LOG.md`.

* Each log entry (bulletpoint with date tag) must include:

  * **What** was done

  * **How** it was done (brief)

  * **Which files** were changed/added/removed

## **2\) Keep the repo clean and easy to navigate**

* Organize and structure repo as per best coding practices.  
* Maintain a **clear, consistent folder structure**.

* Avoid clutter: **do not add unnecessary files or documents**.

* Do not create extra `.md` documentation unless explicitly requested. Use `LOG.md` for history.

## **3\) No shortcuts, no placeholders, no dishonesty**

* Do not:

  * Take shortcuts to “make it work”

  * Add placeholders/stubs **unless explicitly required and approved**

  * Shorten code for brevity at the cost of correctness/clarity

  * “Cook” data, fake results, or hide issues

* Always be accurate and transparent.

## **4\) Investigate problems properly**

* If something breaks or behaves unexpectedly:

  * **Diagnose the root cause**

  * Identify the **specific culprit**

  * Apply the **correct fix**

* Do not bypass the issue by changing direction just to avoid debugging.

## **5\) Review and test before presenting results**

* Before showing anything as “done”:

  * **Run relevant tests** (or add them if missing)

  * **Manually sanity-check** key behavior

  * Ensure it’s **reviewed for correctness** (logic, edge cases, style)

## **6\) Plan-first execution (checklist style)**

* Always, start with a **step-by-step plan**.

* The plan must act as a **checklist**.

* **Update the checklist as steps are completed**.

## **7\) Ask before choosing a different approach**

* If there is uncertainty or you want to propose an alternative approach:

  * **Ask first**

  * Do not proceed with the alternative until approved.

## **8\) When in doubt or unsure due to knowledge cutoff or don't have latest information, use web search to learn or double check**

## **7\) Always do your best, time or tokens are indefinite so ensure you do it right and proper, use skills, plugins and agents to help you whenever appropriate**
